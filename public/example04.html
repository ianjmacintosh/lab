Welp, I guess we just have to live with it and if anyone asks, we'll need to explain how browsers get the last word in how they render our pages.

But wait a second, there's yet another solution. And this one actually makes sense: set the page height using dvh units.

"Device Viewport Height" is the height of the screen, minus the height of the browser chrome. It seems like browser authors have given us exactly what we want.

And even worse, it looks fine when I view the page with Chrome DevTools in "Device Mode" to simulate a mobile screen. Testing a fix is going to require a real phone or an emulator.

Maybe we should respect each browser's authors' vision for how the viewport should work, and if they want to cover the page with their browser chrome, they get the final word.

But before I give up, I do some Googling and find an answer that involves introducing some exotic syntax I've never encountered:

git commit -am "Add `padding-bottom: calc(10px + env(safe-area-inset-bottom))`"

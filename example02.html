Annoyingly, when I view the page with Chrome DevTools in "Device Mode" to simulate a mobile screen, it looks fine. This is a browser issue, so testing a fix is going to require a real phone (or an emulator.)

I begin philosophically musing about how maybe we should respect each browser's authors' vision for how the viewport should work. If they want to cover the page with their browser chrome, so be it.

But before I give up, I do some Googling and find an answer that involves introducing some exotic syntax I've never encountered:

git commit -am "Add `padding-bottom: calc(10px + env(safe-area-inset-bottom))`"
